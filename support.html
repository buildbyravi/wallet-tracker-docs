Quick Start — Option A (easiest)

  1. Click Make a Copy — the Sheet and script wrapper come with it
  2. The library is already linked. Just refresh the sheet — the Balances menu appears
  3. Go to Balances → Setup / Create Dashboard once
  4. Add your wallets in col B (row 3+), chain name in row 1 col C, RPC URL in row 2 col C
  5. Click Balances → Fetch ALL chains — this sheet

Quick Start — Option B (add to existing sheet)

  1. Open your Google Sheet → Extensions → Apps Script
  2. Left sidebar → Libraries + → paste Script ID:
    1yM1l94RPOknc3QMz7l4eRnNRHRFR0vow0sIEmYBh22nZ_kUBbfreBee1
  3. Click Look up → set Identifier to exactly WalletTracker → click Add
  4. Replace everything in Code.gs with this: 
    function onOpen()               { WalletTracker.registerMenu(); }
function onInstall()            { WalletTracker.registerMenu(); }
function wt_fetchCurrentSheet() { WalletTracker.fetchCurrentSheetAllChains(); }
function wt_fetchAllSheets()    { WalletTracker.fetchAllSheetsAllChains(); }
function wt_fetchOneChain()     { WalletTracker.promptFetchOneChain(); }
function wt_setupDashboard()    { WalletTracker.setupDashboard(); }
function wt_addWalletSheet()    { WalletTracker.addWalletSheet(); }
function wt_help()              { WalletTracker.showHelp(); }
  5. Press Ctrl+S → close the tab → refresh your sheet → Balances menu appears
⚠️ The identifier must be WalletTracker exactly — wrong capitalisation causes a ReferenceError.

Sheet layout

  • Col A, row 3+ — Sr No (auto or manual)
  • Col B, row 3+ — wallet addresses (0x…, exactly 42 characters)
  • Row 1, col C+ — chain name (e.g. "Ethereum")
  • Row 2, col C+ — RPC URL (e.g. https://eth.llamarpc.com)

To add another chain: new column → row 1 = name, row 2 = RPC URL. That's it.

⚠️ You only need public wallet addresses (0x…). Never enter private keys or seed phrases.

Large wallet lists — batch mode

Google Apps Script has a 6-minute execution limit. If you have more than ~150 wallets on a single chain, the script auto-splits into batches of 100 and saves progress. Here's how it works:

  • Run Balances → Fetch ALL chains — this sheet as normal
  • After 100 wallets, it saves progress and shows a toast: "Batch 1 done — run Fetch again to continue from row 103"
  • Run the same menu item again — it picks up exactly where it left off
  • Repeat until the toast says "All wallets done"
  • Cells that haven't been fetched yet show — so you can see progress at a glance
💡 For 500+ wallets, use a private RPC key from Alchemy or Infura — public nodes rate-limit heavy traffic.

Getting updates

The library is versioned. How you receive updates depends on how you installed it.

If you used Option A (copied the Sheet)

Your copy has a fixed library version from the day you made the copy. You do not get updates automatically. To update manually:

  1. Open your Sheet → Extensions → Apps Script
  2. Left sidebar → Libraries → click WalletTracker
  3. Change the Version dropdown to the highest available number
  4. Click Save → close the tab → refresh your Sheet

If you used Option B (added the library manually)

Same steps as above. Alternatively, set Version to HEAD (development mode) to always receive the latest deploy automatically — no manual steps needed.

⚠️ HEAD means instant updates. If a broken version is ever deployed, your Sheet will break until it is fixed. A fixed version number is safer if you need stability.

How to know a new version is available

Open the Version dropdown in Apps Script — if a higher number exists than what you have selected, an update is available. Release notes are posted on GitHub Releases.

Common Issues

❌ ReferenceError: WalletTracker is not defined

  • The library isn't linked yet — go to Libraries + and add it first
  • Check the identifier is exactly WalletTracker (not WALLETBALANCETRACKER or anything else)
  • Click the library name in the sidebar → change Identifier → Save

❌ "Cannot connect to RPC endpoint" / RPC error in cell

  • Check the URL in row 2 is correct and starts with https://
  • The RPC provider may be down — check their status page
  • Try a different provider: Alchemy, Infura, dRPC
  • Make sure the RPC matches the chain — an Ethereum RPC won't work for Polygon

❌ Balance not updating

  • Run Balances → Fetch ALL chains — this sheet manually
  • Wallet address must start with 0x and be exactly 42 characters
  • Free public RPCs rate-limit — wait 30 seconds and retry, or use a private key

❌ Wrong balance shown

  • Check you're on mainnet not testnet (or vice versa)
  • RPC nodes can lag — try a different provider
  • Always verify on a blockchain explorer independently

❌ No "Balances" menu

  • Refresh the sheet — the menu only appears after a full page reload
  • Check Apps Script → Code.gs has the 8 wrapper functions pasted in
  • Run onOpen manually from the Apps Script editor once

❌ Script times out on large lists

  • This is expected above ~150 wallets — batch mode handles it automatically
  • Just re-run Fetch ALL chains — this sheet and it resumes from where it stopped

FAQ

Is my data stored on your servers?

No. Everything stays in your Google Sheet. The library runs via Google Apps Script inside your own Google account — no data passes through any server we control.

Which blockchains are supported?

Any EVM-compatible chain with a public RPC — Ethereum, Polygon, BSC, Arbitrum, Optimism, Avalanche, Base, and hundreds more. You configure the RPC, so you control which chains you query.

Can I track multiple wallets?

Yes. Add as many public addresses as you need in column B from row 3 onwards. Batch mode handles lists of any size.

Can I track multiple chains?

Yes. Each column from C onwards is one chain. Add as many columns as you need.

What balances does it show?

Native coin balances — ETH on Ethereum, MATIC on Polygon, BNB on BSC, etc. ERC-20 token balances are not currently supported.

Do I need a paid RPC?

No. Free tiers and public RPCs work fine for personal use. For 100+ wallets or frequent refreshes, a free Alchemy or Infura account gives you higher rate limits.

Is this free?

Yes, completely free. No subscription, no paywalls, no hidden costs.
